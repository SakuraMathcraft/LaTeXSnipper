# LaTeXSnipper Office 插件双击编辑需求

## 1. 目标

实现公式双击编辑，边界如下：

| 宿主         | 公式类型              | 双击编辑 |
| ---------- | ----------------- | ---- |
| Word       | LaTeXSnipper OLE  | 支持   |
| Word       | OMML 公式           | 不支持  |
| Word       | Word 原生公式         | 不支持  |
| PowerPoint | LaTeXSnipper 图片公式 | 支持   |
| PowerPoint | LaTeXSnipper OLE  | 支持   |

---

## 2. 双击入口

### Word OLE

只能通过：

```text
IOleObject::DoVerb
```

触发编辑器。

禁止使用：

```text
鼠标钩子
WM_LBUTTONDBLCLK
双击计时
SelectionChange 模拟双击
```

Word 中双击 OMML、Word 原生公式、普通图片、文本等对象时，不得触发 LaTeXSnipper 编辑器。

### PowerPoint 图片公式

通过：

```text
Application.WindowBeforeDoubleClick
```

处理。

只有满足以下条件才打开编辑器：

```text
单选 Shape
包含 LaTeXSnipper 元数据
RenderEngine == Image
元数据完整
```

符合条件时：

```text
Cancel = true
同步捕获当前 Shape 和 Presentation
打开编辑器
```

普通图片和其他 Shape 保持 PowerPoint 默认行为。

### PowerPoint OLE

PowerPoint 双击事件只负责同步缓存被双击的 OLE Shape，不负责打开编辑器。

随后由：

```text
IOleObject::DoVerb
```

触发编辑器。

不得让 `WindowBeforeDoubleClick` 和 `DoVerb` 各打开一次。

---

## 3. 删除现有错误实现

删除所有通用鼠标双击捕获逻辑，包括：

```text
FormulaDoubleClickWindow 中的鼠标钩子
WH_MOUSE
SetWindowsHookEx
WM_LBUTTONDBLCLK
WmImageActivation
DuplicateWindowMilliseconds
基于时间的重复触发过滤
```

可以保留一个只负责接收 OLE 注册窗口消息的窗口类，例如：

```text
OleActivationMessageWindow
```

该类不得处理任何鼠标消息。

---

## 4. 编辑器实例与公式切换规则

编辑器继续使用单实例。

准确规则：

```text
一个 Word 加载项实例只有一个 MathLive 编辑器窗口
一个 PowerPoint 加载项实例只有一个 MathLive 编辑器窗口
Word 和 PowerPoint 各自独立
```

同一个宿主内，即使打开多个文档或窗口，也共享同一个编辑器窗口。

用户正在编辑公式 A 时，再双击公式 B，必须立即切换到公式 B：

```text
复用现有编辑器窗口
重新 Configure 为公式 B
把当前编辑目标切换为 B
把侧边栏临时预览切换为 B
激活并置前现有编辑器
```

不得继续显示或提交公式 A。

公式切换必须作为一次原子操作完成，以下状态必须同时更新：

```text
当前 Document / Presentation
当前 EquationId
当前 FormulaMetadata
当前 WordFormulaEditTarget / PowerPointFormulaEditTarget
编辑器中的 LaTeX
编辑器的更新模式
侧边栏临时预览
```

不得出现以下中间状态：

```text
编辑器显示 B，但提交目标仍是 A
编辑器显示 A，但侧边栏预览已经是 B
目标已经切换为 B，但编辑器仍提交 A 的内容
```

切换到 B 时，A 在编辑器中的未提交临时修改可以被放弃，不做多公式草稿保存，也不弹出确认框。

Office 文档中的公式 A 本体不得被修改，只有用户提交过的修改才写入文档。

预热创建的隐藏窗口不算正在编辑公式。只要双击得到有效目标，就允许复用该窗口并加载目标公式。

## 5. 编辑目标绑定

双击发生时，必须同步捕获编辑目标。

每次双击新的有效公式，都要用本次捕获的目标替换上一公式的编辑目标。

切换必须发生在加载新公式之前，且不得保留上一公式的提交目标。

不得在异步任务启动后重新读取：

```text
ActiveWindow.Selection
ActiveDocument
ActivePresentation
```

### Word

新增或使用：

```text
WordFormulaEditTarget
```

至少保存：

```text
原始 Document
EquationId
FormulaMetadata
是否为 OLE
```

### PowerPoint

使用：

```text
PowerPointFormulaEditTarget
```

至少保存：

```text
原始 Presentation
EquationId
FormulaMetadata
```

提交时只能更新最后一次双击捕获的目标对象。

用户切换文档、窗口或演示文稿后，提交目标不得改变。

目标被删除或原文档已关闭时，拒绝提交，不得更新当前选择，也不得插入为新公式。

---

## 6. 复制粘贴和身份

复制后的公式必须是独立对象。

```text
原公式 EquationId = A
复制公式 EquationId = B
A != B
```

原公式和副本可以拥有相同初始 LaTeX，但必须可以分别编辑。

`EquationId` 和 `DocumentId` 只保存在 Office 宿主侧元数据中。

OLE payload 不保存：

```text
documentId
equationId
```

OLE payload 只保存公式内容和展示信息。

不得使用 LaTeX、Shape.Name、位置或 OLE payload 作为对象身份。

---

## 7. 不做历史兼容

不需要支持旧文档迁移。

删除并禁止新增：

```text
RemoveJsonProperty
SanitizePayloadIdentity
LegacyPayloadReader
PayloadMigrator
旧 schema fallback
旧字段兼容
旧 payload 自动修复
```

只读取当前格式。

旧对象缺少当前要求的元数据时，视为不可编辑，不进行自动迁移。

---

## 8. Native OLE 要求

`DoVerb` 支持：

```text
OLEIVERB_PRIMARY
OLEIVERB_OPEN
OLEIVERB_SHOW
```

其他 Verb 返回：

```text
OLEOBJ_E_INVALIDVERB
```

`EnumVerbs()` 返回一个：

```text
Edit Formula
```

Native OLE 不创建独立 EXE，不实现 Office 原位编辑器。

`IsRunning()` 返回：

```text
FALSE
```

`GetMiscStatus()` 与安装器注册表必须一致，包含：

```text
OLEMISC_CANTLINKINSIDE
OLEMISC_RENDERINGISDEVICEINDEPENDENT
OLEMISC_NOUIACTIVATE
OLEMISC_IGNOREACTIVATEWHENVISIBLE
OLEMISC_SETCLIENTSITEFIRST
```

不得包含：

```text
OLEMISC_STATIC
```

注册表数值统一为：

```text
672272
```

`DoVerb` 发送激活消息后，不得返回会让 PowerPoint 继续进入 UI 激活流程的结果。

使用：

```text
OLEOBJ_S_CANNOT_DOVERB_NOW
```

## 9. 状态窗格规则

双击编辑不得破坏状态窗格原始草稿。

首次进入公式编辑时：

```text
保存进入公式编辑前的状态窗格原始草稿
临时显示当前公式
```

正在编辑公式 A 时双击公式 B：

```text
保留最初保存的状态窗格原始草稿
把临时预览从 A 直接切换为 B
不得把 A 的临时预览保存成新的原始草稿
```

提交、取消或关闭整个公式编辑会话后：

```text
恢复进入公式编辑前保存的状态窗格原始草稿
```

不得修改 OCR、任务窗格插入和现有草稿恢复语义。

## 11. 验收标准

必须满足：

```text
Word OLE 双击只打开一次编辑器
Word OMML 双击不触发插件
Word 原生公式双击不触发插件
PowerPoint 图片公式双击可以编辑
PowerPoint OLE 双击只打开一次编辑器
普通图片和普通 Shape 保持 Office 默认行为
复制公式后原对象和副本可以分别编辑
编辑 A 时双击 B 会立即切换到 B
切换后编辑器内容属于 B
切换后侧边栏临时预览属于 B
切换后提交目标属于 B
A 的未提交编辑器修改被放弃
公式 A 的文档内容不会被误修改
切换过程中不得出现 A/B 内容与目标串线
切换文档后提交仍更新最后一次双击捕获的目标
目标删除后提交被拒绝
代码中不存在鼠标钩子和双击计时
代码中不存在旧 payload 兼容逻辑
```

完成后输出：

```text
修改文件列表
删除的旧实现列表
Word OLE 双击调用链
PowerPoint 图片双击调用链
PowerPoint OLE 双击调用链
单实例编辑器与公式切换处理说明
侧边栏临时预览切换说明
复制身份处理说明
未执行或未通过的测试
```
