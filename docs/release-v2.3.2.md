# LaTeXSnipper v2.3.2 发布日志

`v2.3.2` 是一次以稳定性、部署体验和界面细节为主的修复版本。重点解决依赖环境、启动流程、历史记录交互、系统托盘和打包清洁度问题。

## 修复

- 修复新版 Qt/PyQt6 组合下 Fluent 下拉框出现玻璃容器效果、圆角与弹出层表现不稳定的问题。
- 修复深色模式下依赖向导勾选框状态不清晰、已安装层与待安装层对齐不一致的问题。
- 修复日志 GUI 在部分环境中无法打开、日志文件被占用时初始化失败的问题，并增加备用日志路径。
- 修复启动阶段 splash 可能重复出现、预热完成后不消失的问题。
- 修复依赖不完整时 splash 置顶遮挡依赖向导和安装进度窗口的问题。
- 修复从设置页进入依赖向导时，功能层验证阶段过早隐藏 splash 造成的启动空档。
- 修复未固定识别结果窗口在下一次截图识别时需要手动关闭的问题；现在新识别开始时会自动隐藏旧结果，识别完成后显示新结果。
- 修复 torch 初始化失败时错误提示过度归因为 VC++ 运行库的问题，区分未安装、二进制依赖加载失败和版本/轮子不匹配等场景。
- 修复系统托盘菜单左侧预留空间过宽、悬停反馈不明显和二级菜单指示样式不协调的问题。
- 修复无历史记录时左侧历史区域被底部长按钮文案撑宽，导致右侧编辑区被挤压的问题。

## 优化

- 优化 PyQt6 与 PyQt6-Fluent-Widgets 的版本矩阵，固定到更稳定的 UI 依赖组合，降低开发模式自修复拉回错误版本的风险。
- 优化依赖安装顺序，GPU 层的 torch、torchvision、torchaudio 与 onnxruntime-gpu 会优先于 pix2text 安装，减少被 CPU 轮子覆盖的概率。
- 优化依赖向导验证逻辑，CORE 层增加 pix2text CPU/ONNX 可用性复核，安装成功与预热成功之间的风险提示更明确。
- 优化私有 Python 3.11 模板，使其更接近绿色、最小、可移植的依赖部署器，并隔离用户目录 site-packages 对打包和安装的污染。
- 优化 PyInstaller 打包规则，清理 bundled python311 中不应随包携带的重型运行时依赖，避免把后续向导安装的包提前塞入安装包。
- 优化启动流程，将模型预热、热键注册等任务延后到主窗口显示之后执行，减少首屏等待。
- 优化启动 splash 生命周期：静默复核依赖时保持显示，只有目录选择、依赖向导或安装进度窗口真正出现前才隐藏。
- 优化历史公式右键菜单顺序为“编辑、复制、收藏、导出为、重命名、删除”，更贴近常用操作路径。
- 优化历史公式列表，每条记录右侧只保留“复制”按钮，收藏和删除移入右键菜单，减少横向拥挤。
- 优化长公式历史记录的换行布局，公式名称与内容分列显示，复制按钮固定在右上角。
- 优化多屏截图识别模式，托盘菜单会根据当前屏幕列表刷新，支持扩展屏选择。
- 优化关闭流程，正常退出时继续保证历史记录与用户配置先保存，再释放热键并退出。

## 升级建议

- 建议 `v2.3.x` 用户升级到 `v2.3.2`，尤其是遇到依赖向导遮挡、下拉框显示异常、torch 错误提示不准确或历史记录区域布局异常的用户。
- 如果升级后依赖状态异常，可从设置页重新进入依赖向导执行强制校验；程序会重新复核已安装功能层并提示需要修复的层。
