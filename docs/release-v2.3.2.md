# LaTeXSnipper v2.3.2 发布日志

## 修复

- 修复深色模式下依赖向导勾选框状态不清晰、已安装层与待安装层对齐不一致的问题。
- 修复新版 Qt/PyQt6 组合下 Fluent 下拉框出现玻璃容器效果、圆角与弹出层表现不稳定的问题。
- 修复日志 GUI 在部分环境中无法打开、日志文件被占用时初始化失败的问题，并增加备用日志路径。
- 修复启动阶段 splash 可能重复出现、预热完成后不消失的问题。
- 修复未固定识别结果窗口在下一次截图识别时需要手动关闭的问题，现在会在新识别开始时自动隐藏，并在识别完成后显示新结果。
- 修复 torch 初始化失败时错误提示过度归因为 VC++ 运行库的问题，区分未安装、二进制依赖加载失败和版本/轮子不匹配等场景。
- 修复系统托盘菜单左侧预留空间过宽、悬停反馈不明显和二级菜单指示样式不协调的问题。

## 优化

- 优化 PyQt6 与 PyQt6-Fluent-Widgets 的版本矩阵，固定到更稳定的 UI 依赖组合，降低开发模式自修复把依赖拉回错误版本的风险。
- 优化依赖安装顺序，GPU 层的 torch、torchvision、torchaudio 与 onnxruntime-gpu 会优先于 pix2text 安装，减少后续被 CPU 轮子覆盖的概率。
- 优化依赖向导验证逻辑，CORE 层增加 pix2text CPU/ONNX 可用性复核，安装成功与预热成功之间的风险提示更明确。
- 优化私有 Python 3.11 模板，使其更接近绿色、最小、可移植的依赖部署器，并隔离用户目录 site-packages 对打包和安装的污染。
- 优化 PyInstaller 打包规则，清理 bundled python311 中不应随包携带的重型运行时依赖，避免把后续向导安装的包提前塞入安装包。
- 优化启动流程，将模型预热、热键注册等任务延后到主窗口显示之后执行，减少首屏等待。
- 优化多屏截图识别模式，托盘菜单会根据当前屏幕列表刷新，支持扩展屏选择。
- 优化关闭流程，正常退出时继续保证历史记录与用户配置先保存，再释放热键并退出。
