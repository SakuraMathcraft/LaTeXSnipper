LaTeXSnipper Office Add-in

Equation entry and screenshot OCR powered by the local LaTeXSnipper Office bridge.

The add-in follows the Office display language. Enable the Office add-in feature in LaTeXSnipper desktop settings and keep the desktop app running before using conversion or screenshot OCR.

Supported Hosts

HostRequirementsDelivered workflow
Word desktop WordApi 1.3 and SharedRuntime 1.1. On Windows, the shared runtime requires Microsoft 365 Word Version 2205 (Build 15202.10000) or Word 2024; on Mac it requires Word 16.61. Tagged editable OMML equations, update/delete, auto or manual numbering, and renumber all.
PowerPoint desktop ImageCoercion 1.1 and SharedRuntime 1.1. The shared runtime requires Microsoft 365 PowerPoint Version 2102 (Build 13722.10000) or PowerPoint 2021/2024 on Windows; PowerPoint 16.46 on Mac. Formula image insertion and numbered formula image insertion.

Web and mobile Office are not product targets because the workflow depends on a local desktop bridge and desktop screenshot capture.

The Windows and macOS runtime installers configure the local https://localhost:8765 site. The Windows installer writes registry entries under both WEF\Developer (auto-load) and WEF\TrustedCatalogs (makes the manifests visible under Shared Folder in Office that support it). The LaTeXSnipper ribbon tab appears on the next start of Word and PowerPoint without extra steps on most configurations. Enterprise administrators can instead deploy the production manifests through Microsoft 365 Integrated apps.

Word Functions

Insert Formula

Converts the current LaTeX to OMML and inserts a new LaTeXSnipper equation at the cursor. Display, automatic numbering, and manual numbering apply to the new equation.

Editor and Load Selected

Editor opens visual entry. Load Selected edits only a LaTeXSnipper equation selected in the document, including selection through its numbered layout.

Delete Selected

Deletes the selected LaTeXSnipper equation and stored source. For a numbered equation it also removes that equation's layout table; for an ordinary equation inside a user table it leaves the user's table intact.

Auto Numbered and Renumber All

Auto Numbered converts one selected unnumbered LaTeXSnipper equation to automatic numbering. Renumber All updates automatic numbers in document order and leaves manual labels unchanged.

A cursor inside an existing LaTeXSnipper equation or numbered layout cannot be used as an insertion point; the add-in reports this instead of allowing Word to raise a raw insertion error.

PowerPoint Functions

Insert Formula writes a PNG cropped to the actual formula bounds. Insert Manual # opens the editor for a user-entered label, then writes one composed PNG containing the formula and label. Screenshot OCR updates the task pane and synchronizes the same LaTeX into an open editor window.

PowerPoint's stable image insertion API does not expose a durable managed identity for an inserted formula image. After an image is deleted, the add-in cannot reliably determine the remaining labels, so it intentionally has no auto numbering, Load Selected, Update, Delete Selected, or Renumber All workflow.

Office.js Capability Boundary

The implementation is complete at the reliable Office.js boundary for this add-in: Word manages equations created and tagged by LaTeXSnipper; PowerPoint receives rendered formula images. No compatibility or guessing logic is kept for unsupported objects.
  • An arbitrary native Word equation has no stored original LaTeXSnipper source and is not loadable or maintainable by this add-in.
  • A copied formula without its document settings is not converted into a managed equation automatically.
  • A PowerPoint inserted image is a slide image, not a native editable LaTeXSnipper equation object.
