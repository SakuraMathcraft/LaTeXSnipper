# LaTeXSnipper v2.0 更新说明

发布日期：2026-03-25

本次 `v2.0` 是一次产品级升级。LaTeXSnipper 不再只是截图识别工具，而是正式进入“识别 + 编辑 + 计算”的桌面数学工作台阶段。

## 主要更新

### 1. 数学工作台正式引入

- 新增独立的“数学工作台”窗口，不再把公式编辑强行塞回主界面。
- 工作台支持单独打开、关闭、拖动和主题切换，不依附主窗口布局。
- 识别结果可一键载入工作台，编辑完成后再写回主编辑器。

### 2. MathLive 公式编辑器接入

- 工作台内嵌 `MathLive math-field`，提供结构化数学输入体验。
- 支持内置 `Math Virtual Keyboard`，适合鼠标点击录入和公式模板编辑。
- LaTeX 输入、实时渲染、MathJSON 展示已经联通。

### 3. 计算能力升级为双引擎架构

- 前端使用 `MathLive Compute Engine` 完成快速计算、化简、数值化等即时操作。
- 对无穷级数、无穷乘积、积分、极限、复杂闭式求值等重表达式，增加本地 `SymPy/mpmath` 高级引擎兜底。
- 本地高级引擎已改为独立子进程执行，避免 CPU 密集计算阻塞主界面。

### 4. 常用计算动作可用

- 支持 `计算`
- 支持 `化简`
- 支持 `数值化`
- 支持 `展开`
- 支持 `因式分解`
- 支持 `求解`

这些动作在前端引擎无效、超时或结果不可靠时，会自动切换到本地高级引擎继续处理。

### 5. 无理根、复根与特殊表达式支持增强

- 求解器补充了 `solve() -> roots() -> nroots()` 多层回退链。
- 因式分解在普通分解无效时，可按根重构因式。
- 对无理根、复根、部分高次多项式场景，结果能力明显强于早期版本。
- 对经典常数类结果，加入“数值近似 -> `nsimplify()` 常数识别”链路，可将部分近似值恢复为闭式结果。

### 6. 结果展示与交互改进

- 计算结果区域支持正确渲染数学结果，而不只是纯文本。
- 对过长结果和复杂表达式，结果区支持滚动查看。
- 工作台内按钮反馈统一使用窗口内 `InfoBar`，不再跑到主窗口提示。
- 内置多组示例，覆盖多项式、方程、积分、无穷级数、无穷乘积、极限、导数等常见场景。

### 7. 工作台主题和体验打磨

- 工作台已经适配浅色/深色主题切换。
- 工具栏重新分组，避免按钮过多时只能全屏使用。
- 虚拟键盘改为更合理的浮层呈现，减少挤压编辑区的问题。
- 状态文本收口到更小的状态栏区域，避免正文区域被提示信息占用。

## 架构变化

`v2.0` 引入了新的编辑器域，相关代码不再继续堆积在 `main.py` 中，新增：

- `src/editor/workbench_window.py`
- `src/editor/workbench_bridge.py`
- `src/editor/advanced_cas.py`
- `src/assets/mathlive/`

这意味着 2.0 之后的公式编辑和计算能力已经具备独立演进基础。

## 兼容性说明

- 当前工作台依赖网络加载 MathLive/Compute Engine 资源，暂未切换为完全本地离线资源方案。
- 老版本“仅识别、不编辑”的使用方式仍可继续使用，但工作台已成为 2.0 的核心能力方向。
- 打包时需要确保 `src/editor/` 与 `src/assets/mathlive/` 资源一并进入产物。

## 建议

- 升级到 `v2.0` 后，优先测试“识别 -> 载入工作台 -> 编辑/计算 -> 写回主编辑器”这条完整链路。
- 重点验证无理根、复根、无穷级数、无穷乘积和复杂因式分解场景。
- 若用于正式发布，建议再做一轮打包版验证，确认高级引擎子进程在冻结环境下工作正常。
