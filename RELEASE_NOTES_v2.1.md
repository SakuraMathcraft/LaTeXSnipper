# LaTeXSnipper v2.1 更新说明

发布日期：2026-03-30

本次 `v2.1` 聚焦于手写识别能力落地。LaTeXSnipper 从“截图识别 + 编辑”继续向更完整的数学输入工作流推进，新增了可独立使用的手写识别窗口，并补齐了与主窗口之间的协作体验。

## 主要更新

### 1. 独立手写识别窗口上线

- 新增独立的“手写识别”窗口，可从主窗口直接打开。
- 手写识别窗口与主窗口解耦，可独立存在、单独最小化，不依附主窗口关闭。
- 识别结果可直接插回主编辑器，也可单独复制 LaTeX。

### 2. 手写画布与工具链落地

- 新增书写、橡皮、圈选修正、清空、撤销、重做等基础工具。
- 支持鼠标与触控笔输入，书写工具、橡皮工具已适配对应光标。
- 当前工具状态提供明确高亮，深色/浅色主题均可识别。

### 3. 画布交互显著增强

- 画布支持自动延展，并改为滚动画布模型，不再局限于固定首屏区域。
- 支持 `Ctrl + 滚轮` 缩放画布视图。
- 支持关闭自动聚焦后使用鼠标右键拖动画布。
- 修复缩放后继续书写导致画布逻辑尺寸异常、露白的问题。

### 4. 手写识别结果联动预览

- 停笔后自动调用现有 `pix2text` 识别引擎。
- 右侧提供 `LaTeX 结果` 与 `实时预览` 双区域。
- 混合内容预览、结果区选中颜色、深色模式适配均已补齐。

### 5. 主窗口与手写识别互斥保护

- 主窗口截图识别、PDF 识别与手写识别现在共享忙碌保护，不再并发撞同一个 `pix2text` worker。
- 识别忙碌时统一改为轻量 `InfoBar` 提示，不再弹阻塞式对话框。

### 6. 主界面入口与布局整理

- 主工具栏调整为更合理的识别入口顺序：图片识别、PDF 识别、手写识别。
- 移除低频的顶栏收藏入口，收紧主界面按钮布局。
- 主窗口版本号与打包资源已同步更新到 `v2.1`。

## 架构变化

`v2.1` 新增手写识别模块，相关实现已从主窗口中拆出，新增：

- `src/handwriting/handwriting_window.py`
- `src/handwriting/ink_canvas.py`
- `src/handwriting/stroke_store.py`
- `src/handwriting/recognizer.py`
- `src/handwriting/tools.py`
- `src/handwriting/types.py`

打包配置也已同步补充手写识别目录，确保冻结产物包含完整功能。

## 兼容性说明

- 手写识别当前仍复用 `pix2text` 作为后端识别引擎，因此复杂手写公式的准确率仍受图像识别模型能力限制。
- 橡皮与圈选修正的几何裁剪仍运行在界面线程，超大笔迹量场景下可能出现性能下降。
- 手写识别窗口依赖现有 `pix2text` 运行环境，发布包需要继续携带当前 Python 依赖体系。

## 建议

- 升级到 `v2.1` 后，优先测试“手写 -> 自动识别 -> 手动修正 -> 插回主编辑器”这条链路。
- 重点验证缩放后书写、自动延展、橡皮擦与圈选修正的连续操作体验。
- 正式发布前建议用打包版再回归一轮主窗口识别与手写识别的互斥逻辑。
