# LaTeXSnipper v1.06 更新说明

发布日期：2026-03-25

本次 `v1.06` 重点围绕启动体验、截图识别链路、多屏支持、主题切换适配和更新体验做了系统性优化，属于一次以可用性与稳定性为核心的版本更新。

## 主要更新

### 1. 启动性能优化

- 启动阶段的模型预热调整为后台异步执行，主窗口可更快进入可用状态。
- 启动提示窗口文案改为反映真实流程，避免“正在初始化模型”这类过时提示。
- 修复高 DPI 下启动窗口画布尺寸异常的问题，消除四周多余留白。
- 优化状态栏模型状态展示，区分“预热中”“已加载”“未就绪”等阶段。

### 2. 扩展屏与高分屏截图修复

- 修复多屏/扩展屏环境下的截图区域映射问题。
- 修复高 DPI 缩放场景下截图坐标被重复换算的问题。
- 截图链路改为使用正确的逻辑坐标，减少截偏、截错区域导致的识别错误。
- 提升公式识别前输入图像的准确性，改善“框选正确但识别内容明显不对”的问题。

### 3. 主题切换与深色模式适配

- 补齐主窗口内部分提示文案、公式标签提示文字的深浅色适配。
- 检查更新窗口增加主题切换支持，深色模式下背景、正文、链接和提示文字可正常阅读。
- 依赖环境选择窗口适配深色模式，并统一按钮、复选框高亮色到主窗口主题体系。
- 多处原先固定浅色样式的界面元素已改为跟随当前主题动态切换。

### 4. 交互体验优化

- 快捷键设置窗口改为独立非阻塞窗口，交互方式与设置窗口保持一致。
- 快捷键设置成功、格式错误、历史记录为空、收藏夹为空等提示统一改为 `InfoBar`，减少阻塞式弹窗打断。
- 收藏夹为空时的提示会显示在主窗口，避免在子窗口内提示位置不一致。

### 5. 检查更新增强

- 检查更新窗口支持直接下载 GitHub Release 资产，不再只能跳转到网页手动下载。
- 下载过程中可在窗口内显示进度条和百分比。
- 下载完成后增加 SHA256 计算与 Windows 安装器签名状态检查。
- 启动安装前增加确认提示；在打包版环境下可拉起安装器并退出当前程序。

### 6. 识别运行时与状态链路修复

- 修复后台预热完成后状态未正确回到主线程更新的问题。
- 避免同一启动流程中重复触发 warmup 导致的重复日志。
- 首次识别前若模型仍在后台预热，界面会给出更明确的状态反馈。
- `pix2text` 的预热状态与 worker 建立过程现在能够更准确地反映到界面状态上。

## 兼容性说明

- 当前版本继续使用 `pix2text` 作为唯一识别模型族。
- 依赖层结构仍为 `BASIC / CORE / HEAVY_CPU / HEAVY_GPU`，无需额外迁移。
- 旧版本设置中的部分 `v1.05` 残留提示文案已清理，不影响现有配置文件使用。

## 建议

- 升级到 `v1.06` 后，建议在扩展屏或高分屏环境下重新测试一次截图识别。
- 如使用深色模式，可重点检查更新窗口、依赖选择窗口和主界面提示文字是否符合预期。
- 如程序内检测到新版本，建议直接使用内置下载与安装流程完成更新。
