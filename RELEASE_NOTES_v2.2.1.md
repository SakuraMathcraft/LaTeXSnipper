# LaTeXSnipper v2.2.1 更新日志

发布日期：2026-04-03

`v2.2.1` 是一次以体验优化和交互收口为主的小版本更新，不扩功能范围，主要集中在：

- 手写识别交互稳定性
- 外部模型设置与状态反馈
- 识别结果窗口操作便捷性
- 历史记录浏览体验

## 本版重点

1. 手写识别体验优化

- 重构手写画布交互模型：
  - 左键只在安全书写区内工作
  - 缓冲区禁止左键书写
  - 右键负责主动拖动画布
- 安全书写区与缓冲区改为基于当前可见视口动态计算
- 缓冲区增加可视化提示，并随窗口大小、缩放、滚动和分栏比例动态刷新
- `Ctrl + 滚轮` 改为以鼠标指针为锚点缩放，提升局部放大/缩小定位体验
- 修复最大缩放附近的视图漂移、误滚动和缓冲区显示异常
- 自动聚焦与右键拖动画布不再冲突：
  - 可先手动拖到合适位置书写
  - 停笔稳定后再自动回到内容重心
- 自动聚焦触发时机延后，避免连续书写过程中抢视图
- 手写窗口“插入”后不再关闭窗口，当前内容会保留，便于继续修改
- 去除手写窗口底部“取消”按钮，降低误触概率

2. 手写结果展示优化

- 手写结果预览支持多行混合内容渲染
- 文本行与数学行按启发式规则分块显示
- `latex` 模式下按行去除多余的外层 `$$ / \[ \] / \(...\)` 包裹
- 预览区改为文档流布局，长内容可滚动查看

3. 外部模型设置与反馈优化

- 设置页外部模型状态改为动态三态：
  - 已配置，尚未测试连接
  - 已连接
  - 连接失败
- 测试连接结果与当前配置签名联动，避免状态显示失真
- `GLM-OCR` 预设调整为更贴近实际使用的默认配置：
  - 默认协议：`Ollama`
  - 默认地址：`http://127.0.0.1:11434`
  - 默认输出偏好：`latex`
- 外部模型帮助文档同步当前真实逻辑，避免 PDF / 提示词说明与实际行为不一致

4. 识别结果与历史记录体验优化

- 识别结果窗口新增直接导出入口，不必回主窗口再操作
- 识别结果窗口中的导出提示改为在当前窗口内弹出
- 历史记录新增顺序切换：
  - `最新在前`
  - `最早在前`
- 倒序浏览下的编辑、删除、加载逻辑已同步修正，避免错位操作

## 其它修复

- 修复手写识别线程关闭时偶发的 `QThread: Destroyed while thread is still running`
- 修复手写错误提示中重复句号的问题
- 修复外部模型状态文案和帮助文本中的过时描述
- 收短外部模型连接失败提示，避免把大段底层异常直接弹给用户
- 更新若干日志文案，使 `external_model`、`pix2text`、真实模型名的显示更准确

## 说明

- `v2.2.1` 仍延续 `v2.2` 的能力边界：
  - 外部模型支持截图 / 图片 / 手写 / PDF 识别接入
  - `MinerU` 仍需以实际部署接口为准做联调验证
- 本版没有新增新的识别后端类型，主要目标是把已有交互打磨到可长期使用的状态
