# LaTeXSnipper v2.2 发布说明
<img width="1518" height="1020" alt="v2 2宣传" src="https://github.com/user-attachments/assets/79fbce2c-948f-4321-a314-18ec9e1a409c" />

发布日期：2026-04-01

v2.2 正式版计划范围已完成，外部模型识别链路已接入主程序。

## 本版已完成

1. 外部模型全链路接入

- 图片 / 截图识别支持 `external_model`
- PDF 识别支持外部模型分流，包含进度、取消、结果聚合
- 手写识别支持外部模型分流

2. 文档解析能力扩展

- 新增文档聚合管线 `document_pipeline`
- PDF 外部模型流程固定为整份文档恢复
- PDF 入口支持导出格式选择与 DPI 滑动条调节
- PDF 导出模板重写，Markdown 保留结构层级，LaTeX 模板补齐 `ctex`、图表和 `tikz` / `tikz-cd` 支持

3. 协议与配置能力

- 支持 `Ollama`、`OpenAI-compatible`、`MinerU`
- 设置页支持协议化配置、推荐预设、测试连接、教程说明
- 按协议动态校验必填项，`MinerU` 不再误报“模型名必填”
- `GLM-OCR` 预设默认按 `Ollama + http://127.0.0.1:11434` 配置，默认输出偏好为 `latex`

4. 结果统一与兼容

- 外部模型结果统一回流到确认、历史、收藏、预览流程
- 继续复用现有 `pix2text` 渲染类型体系
- 识别日志改为显示真实外部模型名，而不是固定显示 `external_model`

## 关键优化与修复

1. 设置页与说明文案收口

- 明确区分图片 / 截图 / 手写与 PDF 的提示词生效范围
- 去除过时说明，避免误以为设置页模板会直接决定 PDF 模板
- “查看说明”改为独立窗口，减少阻塞式弹窗干扰

2. 外部模型连接与报错体验

- 测试连接改为后台线程执行，避免设置窗口卡顿
- 测试连接补充模型名校验，提前发现 `model not found`
- HTTP 错误提示收短，避免直接弹出整段底层异常

3. PDF 外部模型稳定性

- 移除“论文 / 期刊模板”选择，统一按文档恢复逻辑处理
- 移除 PDF 偏好模板选择，固定使用通用文档解析提示词
- PDF 渲染分辨率改为 DPI 滑动条，并给出外部模型推荐区间
- 清理常见多模态控制 token，如 `<|begin_of_image|>`，减少污染结果

4. Prompt 与导出收口

- 精简通用提示词模板，保留基础模板与文档恢复模板
- `ocr_document_page_v1` 聚焦 PDF 解析、数学公式与交换图保留
- Markdown 导出统一图片占位标记：
  - `[IMAGE_PLACEHOLDER: 描述]`

## 当前说明

- 外部模型是否使用 GPU 由服务端决定，如 `Ollama`、`MinerU`、`OpenAI-compatible` 服务；客户端不直接控制推理设备。
- 图片 / 截图 / 手写识别会使用设置页中的输出偏好、提示词模板和自定义提示词。
- PDF 识别的导出格式在 PDF 入口单独选择；默认固定走通用文档恢复提示词。
- 若填写了自定义提示词，`OpenAI-compatible / Ollama` 的 PDF 识别也会优先使用它。
- `MinerU` 原生接口已接入客户端，实际效果仍取决于服务端接口形状与部署情况。
- v2.0 以后的用户可以直接在设置页面下载更新包。
