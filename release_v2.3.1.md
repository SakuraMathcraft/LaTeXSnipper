# LaTeXSnipper v2.3.1 发布说明
<img width="1564" height="978" alt="v2 3 1" src="https://github.com/user-attachments/assets/a6c3d531-e6f8-4b91-9084-a9a2fa18c1ed" />

<mark>新增双语阅读入口，支持 PDF 页级中英对照阅读，并补充自动排版窗口搜索、容错编译与翻译后端扩展能力<mark>

`v2.3.1` 是一次承接 `v2.3` 自动排版工作流的小版本更新，重点不是再扩一整套重功能，而是把“阅读修订”这一环真正接上，同时把自动排版窗口的编辑、搜索、编译容错体验继续补齐，并为后续在线翻译接口扩展预留统一切换能力。

## 本版重点：双语阅读

- 主窗口新增 `双语阅读` 入口
  - 位置放在 `数学工作台` 前
  - 打开独立的 PDF 双语阅读窗口
- 支持页级原文 / 中文对照阅读
  - 左侧查看 PDF
  - 右侧显示当前页原文与中文译文
- 默认接入 `Argos Translate`
  - `BASIC` 层新增 `argostranslate`
  - 双语阅读窗口内可直接检测 / 安装英译中模型包
- 当前页翻译支持缓存与后台预取
  - 已翻译页面可直接复用
  - 后续页面可静默预取，减少下翻等待
- 补充在线翻译接口切换能力
  - 新增 `Azure Translator`
  - 新增 `Google Cloud Translation`
  - 新增 `DeepL API Free`
  - 接口配置入口直接放在双语阅读窗口内，方便按需切换
- 接口配置体验优化
  - 配置窗口内补充官网申请地址，方便用户申请 `API Key`
  - 在线接口错误提示细化为：
    - `401 / 403`：鉴权失败 / 密钥错误
    - `429`：限流 / 免费额度耗尽
    - `5xx`：服务端异常

## 自动排版窗口补强

- 自动排版窗口新增源码搜索条
  - 支持 `Alt+F` 打开搜索
  - 支持 `F3 / Shift+F3` 前后跳转
- 编译策略优化
  - 编译错误不再直接阻断 PDF 预览
  - 有新 PDF 就更新，没有新 PDF 就保留上一版
  - 错误与警告单独在日志区展示
- 日志区交互优化
  - 打开公式编辑器时自动隐藏
  - 关闭编辑器后恢复显示
  - 支持拖拽拉伸

## 小优化 / 错误修复

- 双语阅读窗口去除与当前场景无关的按钮，减少干扰
- 安装完英中模型后自动隐藏安装按钮，释放顶部空间
- 调整 PDF 预览初始宽度，占窗口更合理比例
- 修复双语阅读窗口重复打开偶发闪退问题
- 修复翻译线程未安全回收时的 `QThread: Destroyed while thread is still running`
- 修复翻译过程中阻塞 UI、关闭窗口闪退的问题，改为安全回收后台资源
- 修复当前页译文在翻页后短暂显示旧内容的问题
- 去除打开双语阅读窗口时不必要的提示弹窗与预览引擎提示噪音
- 更新窗口改为尽量使用原生 `Fluent / Qt` 主题，仅保留日志 HTML 最小样式
- 全局截图热键在存在模态 / 阻塞窗口时会直接返回，并轻提示先关闭对话框，避免误触导致界面卡死
- 修复开发模式下误复用 `dist\\LaTeXSnipper\\_internal\\deps` 私有解释器导致的重启异常

## 安装包与部署调整

- 继续收紧打包内容，`_internal` 只尽量保留首启 UI 与引导所需的最小运行时
- 业务依赖继续交由依赖向导按功能层部署，避免在安装包内重复携带
- 清理一批与功能层重复的打包内容
  - 包括部分 `numpy / fitz / matplotlib / latex2mathml / lxml` 等运行库
  - 同时移除若干重复进入包内的根目录 `.py` 数据副本
- 保留用户预留的私有 Python 环境 `deps`，不影响首次启动后的完整部署流程

## 升级建议

- `v2.3` 用户可直接升级到 `v2.3.1`
- 如果要使用双语阅读翻译能力：
  - 先打开依赖向导，程序会自动检测出缺失的翻译模块
  - 安装并验证完毕后重启程序
  - 再在双语阅读窗口里安装 `Argos` 英译中模型
- 如果要使用在线翻译接口：
  - 在双语阅读窗口中切换到目标引擎
  - 按提示填写对应平台申请的 `API Key`
  - 建议优先使用带免费额度的官方接口进行测试
