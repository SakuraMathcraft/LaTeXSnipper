# LaTeXSnipper v2.2 Beta 更新说明

发布日期：2026-03-31

本次 `v2.2 beta` 聚焦于“外部模型识别”能力的首轮落地。LaTeXSnipper 在保留现有 `pix2text` 内置识别链路的同时，新增了可配置的外部模型入口，用于接入本地或部分在线多模态 OCR / VLM 服务。

这是一个 Beta 版本，目标是先把“可接入、可配置、可真实联调”的主链路跑通，而不是一次性覆盖所有文档解析场景。

## 主要更新

### 1. 设置页新增“外部模型...”入口

- 识别模型下拉框新增 `外部模型...`。
- `pix2text` 仍保留为内置识别模型。
- 外部模型作为独立路径接入，不影响现有 `pix2text` 主体实现。

### 2. 新增独立外部模型模块

新增目录：

- `src/backend/external_model/`

当前已包含：

- `client.py`
- `worker.py`
- `prompts.py`
- `presets.py`
- `schemas.py`
- `errors.py`

这意味着外部模型请求、配置、预设、结果解析和后台线程逻辑已经从主窗口代码中分离出来，后续可单独演进。

### 3. 首轮支持两类协议

当前支持：

- `Ollama`
- `OpenAI-compatible`

其中：

- `Ollama` 适合本地部署的视觉模型，例如 `qwen2.5vl`
- `OpenAI-compatible` 适合接入兼容 `/v1/chat/completions` 的本地或线上服务

### 4. 外部模型设置引导上线

设置页中的外部模型配置区已支持：

- 协议选择
- `Base URL`
- 模型名
- `API Key`
- 超时
- 输出偏好
- 提示词模板
- 自定义提示词
- 推荐预设

并新增了独立的“外部模型配置说明”窗口，说明内容包含：

- 本地 / 线上接口说明
- 必填 / 选填字段说明
- 本地 Ollama 示例
- 常见错误说明

### 5. 测试连接支持后台执行

- “测试连接”已改为后台线程执行，不再阻塞设置窗口。
- 测试按钮在执行期间会进入 `测试中...` 状态。
- 当前已有测试任务运行时会阻止重复触发。

### 6. 测试连接已补模型名校验

“测试连接”不再只是检查接口是否存活，还会验证模型名是否真实存在：

- `Ollama` 会读取 `/api/tags`
- `OpenAI-compatible` 会读取 `/v1/models`

如果模型名填写错误，现在会在测试阶段直接提示，而不是等到正式识别时才失败。

### 7. 图片 / 截图识别已支持外部模型

当前外部模型已经接入主窗口图片识别链路：

- 设置页切换到 `外部模型...`
- 测试连接通过
- 截图或图片识别可直接走外部模型接口

这条链路已通过 `Ollama + qwen2.5vl:3b` 的本地联调验证。

## 本版本范围说明

### 已完成

- 外部模型设置入口
- 外部模型配置保存
- 教程窗口
- 后台测试连接
- 模型名校验
- `Ollama` 协议接入
- `OpenAI-compatible` 协议接入框架
- 主窗口截图 / 图片识别接入外部模型

### 暂未纳入 v2.2 Beta

以下内容明确放到后续版本，不属于当前 `v2.2 beta` 范围：

- PDF 识别接管
- 整页文档解析接管
- MinerU 原生接口适配
- 外部模型接管手写识别全链路
- 文档级 OCR / 版面恢复 / 表格提取专项流程

也就是说，当前 `v2.2 beta` 的外部模型能力应理解为：

`图片 / 截图识别增强`

而不是：

`完整文档解析替代方案`

## 兼容性说明

- `pix2text` 仍保留为内置识别模型。
- 从外部模型切回 `pix2text` 时，当前改为“首次识别时再加载”，以降低切换时的崩溃风险。
- `OpenAI-compatible` 虽然在技术上支持线上服务，但本版本仍以本地接口联调为主。
- 不同外部模型对提示词、输出风格、响应速度差异较大，当前仍需要按实际服务情况调试。

## 建议

- Beta 阶段优先使用 `Ollama + qwen2.5vl:3b / 7b` 做本地联调。
- 外部模型首次使用时，先完成：
  1. 应用预设
  2. 校正模型名
  3. 测试连接
  4. 再做截图识别
- 正式发布前，建议继续补一轮回归验证，重点覆盖：
  - `external_model -> pix2text` 切换稳定性
  - 外部模型识别结果在确认弹窗、历史记录中的行为
  - 至少一类 `OpenAI-compatible` 服务的真实联调
