LaTeXSnipper Office 插件

在 Word 和 PowerPoint 中插入、编辑和维护由 LaTeXSnipper 管理的原生公式。

系统要求

组件要求
Windows 版本Windows 10 22H2 或更高版本 / Windows 11
Office 版本
  • Microsoft 365 应用（当前通道或月度企业通道）
  • Office 2024 / 2021 / 2019 零售版或批量许可版
  • Office LTSC 2024 / 2021
运行时.NET Framework 4.8；Microsoft Edge WebView2 Runtime（Windows 11 已内置；M365 会自动安装）
Office 体系仅支持 32 位和 64 位桌面版 Office；不支持网页版、移动版和 macOS 版 Office
LaTeXSnipper 桌面端需在本机运行并开启 Office 插件功能，插件通过 127.0.0.1:28765 与本机 Bridge 通信

快速开始

1. 启动桌面端

打开 LaTeXSnipper 桌面端，在设置中启用 Office 插件功能。

2. 编写公式

在侧边栏输入 LaTeX。截图识别结果会自动写回。

3. 插入到文档

使用 Ribbon 按钮或侧边栏的插入按钮。Word 写入原生 OMML，PowerPoint 插入高清图片。

如果请求失败，请确认桌面端正在运行且 Office 插件功能已开启。

Word 功能

原生 OMML 公式

通过本机 Bridge 将 LaTeX 转换为 Word 原生 OMML。行内公式跟随正文；行间公式居中；带编号公式以表格布局保存公式和编号。

加载、更新和删除

选中受管理公式后可加载到编辑器修改。更新保留元数据；删除同时移除公式和元数据。

自动编号与重编号

带编号公式支持自动编号，重编号按文档顺序更新所有编号文本，不重新渲染公式。编号位置（左/右）可在设置中选择。

截图识别

点击后等待桌面端截图结果，再次点击可取消。

PowerPoint 功能

高清公式图片

通过本机 Bridge 将 LaTeX 渲染为高 DPI 的 PNG 图片插入幻灯片。公式元数据保存在形状标签中。

加载和删除

选中受管理公式后可加载到独立编辑器修改，修改后新公式替换原位置旧公式。删除同时移除形状和元数据。

截图识别

点击后等待桌面端截图结果，识别后的 LaTeX 自动填入编辑器。再次点击可取消。

插件架构

项目实现
运行方式Windows 原生 VSTO 插件，Ribbon、侧边栏、设置和帮助页均在 Office 内打开。
Word 公式写入原生 OMML 公式，附带 LaTeX 源码、显示模式、编号模式等管理元数据。
PowerPoint 公式插入高 DPI PNG 图片到幻灯片，元数据写入形状标签（shape.Tags）。
连接方式HTTP 连接本机 Bridge（127.0.0.1:28765），LaTeX 转换和截图识别均由桌面端完成。
编辑器WebView2 托管的 MathLive 编辑器，支持可视化编辑和实时预览。

边界说明

  • 只有由 LaTeXSnipper 插入并保留元数据的公式才能被插件管理。
  • 普通 Office 公式没有 LaTeXSnipper 元数据，插件不会猜测其 LaTeX 来源。
  • 复制粘贴后元数据丢失的公式无法继续维护。
  • 如果加载项未显示，请关闭 Office，重新运行安装包进行修复或重新安装，然后再打开 Office。

LaTeXSnipper Office Plugin

Insert, edit, and maintain native formulas managed by LaTeXSnipper in Word and PowerPoint.

System Requirements

ComponentRequirement
WindowsWindows 10 22H2 or later / Windows 11
Office
  • Microsoft 365 Apps (Current or Monthly Enterprise Channel)
  • Office 2024 / 2021 / 2019 Retail or Volume License
  • Office LTSC 2024 / 2021
Runtime.NET Framework 4.8; Microsoft Edge WebView2 Runtime (built into Windows 11; auto-installed by M365)
Office Platform32-bit and 64-bit Windows desktop Office only; web, mobile, and macOS are not supported
LaTeXSnipper DesktopMust be running locally with the Office plugin feature enabled; communicates via local Bridge at 127.0.0.1:28765

Quick Start

1. Start the client

Open LaTeXSnipper and enable Office plugin in settings.

2. Prepare LaTeX

Type LaTeX in the side pane. Screenshot OCR writes back automatically.

3. Insert into Office

Use the Ribbon or side pane. Word writes native OMML; PowerPoint inserts high-resolution images.

If requests fail, make sure the desktop client is running and Office plugin is enabled.

Word Features

Native OMML Formulas

Converts LaTeX to native Word OMML via the local Bridge. Inline formulas flow with text; display formulas are centered; numbered formulas use a table layout.

Load, Update, Delete

Managed formulas can be loaded into the editor. Updates preserve metadata; deletion removes both formula and metadata.

Auto Numbering & Renumber

Numbered formulas support automatic numbering; renumbering updates all numbers in document order without re-rendering. Number placement (left/right) is configurable in Settings.

Screenshot OCR

Click to wait for the next desktop screenshot; click again to cancel.

PowerPoint Features

High-DPI Formula Images

Converts LaTeX to high-DPI PNG images via the local Bridge and inserts them onto slides. Metadata is stored in shape tags.

Load and Delete

Managed formulas can be loaded into the standalone editor. After editing, the new formula replaces the original at the same position. Deleting removes both shape and metadata.

Screenshot OCR

Click to wait for the next desktop screenshot; recognized LaTeX is loaded into the editor. Click again to cancel.

Plugin Architecture

AreaImplementation
RuntimeNative Windows VSTO add-in. Ribbon, side pane, settings, and help open inside Office.
Word OutputNative OMML with LaTeX source, display mode, and numbering metadata.
PowerPoint OutputHigh-DPI PNG images inserted onto slides; metadata stored in shape tags.
ConnectionHTTP to local Bridge at 127.0.0.1:28765; LaTeX conversion and OCR run on the desktop client.
EditorWebView2-hosted MathLive editor with visual editing and live preview.

Boundaries

  • Only formulas inserted by LaTeXSnipper with metadata can be managed by the plugin.
  • Plain Office equations lack LaTeXSnipper metadata; the plugin does not guess their LaTeX source.
  • Formulas that lose metadata after copy-paste cannot be managed.
  • If the add-in does not appear, close Office, run the installer again to repair or reinstall the add-in, then reopen Office.
